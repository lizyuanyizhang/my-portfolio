# 可视化写作 + 自动上线方案对比

> 在可视化工具里排版内容，网站自动更新，无需改代码。

---

## 一、方案总览

| 方案 | 可视化程度 | 与现有站点关系 | 上手难度 | 上线成功率 |
|------|------------|----------------|----------|------------|
| **A. Notion + Super.so** | ⭐⭐⭐⭐⭐ | 独立建站（可替代文字页） | 极低 | 很高 |
| **B. 语雀 + Elog** | ⭐⭐⭐⭐ | 同步到现有项目 | 低 | 高 |

## 二、方案 A：Notion + Super.so（最省心） ✅ 已选用

> **配置说明**：见 [NOTION_SUPER_SETUP.md](./NOTION_SUPER_SETUP.md)。  
> 在 `personalInfo.essaysExternalUrl` 填入 Super 站点 URL 后，导航栏「文字」会跳转到该外链。

### 适合谁

- 只想在「白板」里写东西，不想碰代码
- 接受文字板块用另一个子站展示（或整体迁移到 Notion 站）

### 怎么做

1. 注册 [Notion](https://notion.so)（免费）
2. 在 Notion 里建一个「文字」页面，像文档一样写、排版、插入图片
3. 右上角「Share」→ 开启「Share to web」
4. 注册 [Super.so](https://super.so)，新建站点，填入 Notion 页面链接
5. 得到 `xxx.super.site`，就是你的「文字站」

### 与你现有站点的关系

- **方式 1**：把导航栏「文字」的链接改成 `https://你的文字站.super.site`，点击跳转过去
- **方式 2**：直接整体用 Notion + Super 建站，放弃当前 React 站点

### 优点 / 缺点

| 优点 | 缺点 |
|------|------|
| 全程可视化，零代码 | 风格与现有站可能不一致 |
| 免费（自定义域名需付费） | 文字是独立子站，不是同一域名下 |

## 三、方案 B/C/D：语雀 / Notion / FlowUs + Elog（保留现有站点）

### 适合谁

- 想保留现在的 React 站点
- 愿意在 Notion / 语雀 / FlowUs 里写作，再自动同步到网站

### 通用流程

1. **选写作平台**：语雀 / Notion / FlowUs 任选一个
2. 在平台里写文章，支持富文本、图片、表格
3. **配置 Elog**：在项目里安装 [Elog](https://github.com/LetTTGACO/elog)，填写平台账号和 Token
4. **同步**：运行 `elog sync`，把文章导出成 Markdown
5. **接入你的站**：把导出的 Markdown 转成你站点用的数据格式（如 `essays` JSON），或写个脚本自动更新 `data.zh.json`

### 三个平台对比

| 平台 | 特点 | 适合 |
|------|------|------|
| **语雀** | 国产、中文、团队协作 | 习惯中文产品 |
| **Notion** | 块编辑、数据库、模板多 | 喜欢 Notion 生态 |
| **FlowUs 息流** | 国产 Notion 替代、中文 | 想用类似 Notion 的国产工具 |

### 需要做的开发

- 配置 Elog（约 10–20 分钟）
- 写一个小脚本：把 Elog 输出的 Markdown 转成你站点的 essays 格式
- 或用 GitHub Actions 自动化：推送后自动同步并部署

## 四、方案 E：TinaCMS（在站内可视化编辑）

### 适合谁

- 希望「编辑入口」就在自己网站里
- 可以接受一些技术配置（Next.js / 静态站点等）

### 怎么做

- Tina 提供可视化编辑器，直接在你的网页上改内容
- 改完写入 Git，触发 Vercel 等重新部署
- 需要把项目改造成支持 Tina 的架构（如 Next.js + Tina）

### 优点 / 缺点

| 优点 | 缺点 |
|------|------|
| 编辑和展示都在同一站点 | 需要做架构迁移 |
| 可视化编辑体验好 | 配置相对复杂 |

## 五、推荐选择

### 想最少折腾：选 **Notion + Super.so（方案 A）**

- 在 Notion 里写和排版
- Super 一键变成网站
- 把现有站「文字」链接改成 Super 站的地址即可
- 不做任何代码修改

### 想保留现有站点：选 **语雀 + Elog（方案 B）**

- 语雀：纯中文、可视化写作
- Elog：开源、社区成熟
- 需：配置 Elog + 一个小脚本把导出内容转成 essays 格式

### 已有 Notion 习惯：选 **Notion + Elog（方案 C）**

- 和语雀方案类似，只是写作平台换成 Notion
- 项目里已有 Notion 配置，后续可考虑直接对接 Notion API，减少 Elog 环节

---

## 六、快速决策

```
只想可视化写作、尽快上线？
→ 选 A：Notion + Super，把「文字」链到新站

要保留现有 React 站点，又想要可视化写作？
→ 选 B 或 C：语雀/Notion + Elog，再写脚本接到 essays
```

如需具体配置步骤，可指定方案（A/B/C）再细化。
