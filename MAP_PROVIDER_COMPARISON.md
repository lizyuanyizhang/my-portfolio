# 地图服务选型：高德 vs Google Maps

> 本项目当前使用高德地图。若服务器在境外（如 GitHub Pages 美国节点），可评估切换到 Google Maps。

---

## 一、高德地图 (Amap) vs Google Maps

| 对比项     | 高德地图 | Google Maps |
|------------|----------|-------------|
| **免费额度** | 约 30 万次/天（JS API） | 约 1 万次/月（Dynamic Maps） |
| **国内访问** | 快、稳定 | 需代理，不稳定 |
| **境外访问** | 可能较慢、易超时 | 快、稳定 |
| **白名单** | 需配置域名 | 可限制 HTTP Referrer |
| **安全密钥** | 2021 年后申请的 Key 必须配置 | 无需 |
| **文档语言** | 中文 | 英文为主 |

---

## 二、Google Maps 收费（2024–2025）

### 免费额度

- **Dynamic Maps**：每月约 **10,000 次** 地图加载免费
- 每 1 次“加载地图”≈ 1 次计费（如用户打开带地图的页面）
- 2025 年 3 月起改为按品类免费额度，约等同于每月 $3,250 等值免费用量

### 超额收费（Dynamic Maps）

| 月用量       | 每 1000 次费用 |
|--------------|----------------|
| 1 万–10 万   | $7.00          |
| 10 万–50 万  | $5.60          |
| 50 万–100 万 | $4.20          |

### 个人站点用量估算

- 假设每日 100 次访问时间轴页（含地图）≈ 约 3,000 次/月
- 在 **1 万次/月** 免费额度内，通常 **无费用**
- 需在 Google Cloud 开启计费（绑定信用卡），但未超额不会扣费

---

## 三、何时考虑换 Google Maps

| 场景                     | 建议 |
|--------------------------|------|
| 高德经常超时、加载失败   | 可尝试 Google Maps |
| 主要为国内用户           | 继续用高德 |
| 主要为境外用户           | 更适合 Google Maps |
| 希望减少白名单/密钥配置  | Google Maps 配置更简单 |

---

## 四、切换需改动

若要接入 Google Maps，需：

1. 在 [Google Cloud Console](https://console.cloud.google.com/) 创建项目并启用 Maps JavaScript API
2. 安装 `@react-google-maps/api` 或直接用官方 JS API
3. 重写 `LocationTrailMap` 的渲染逻辑（API 不同）
4. 在 GitHub Secrets 中配置 `VITE_GOOGLE_MAPS_API_KEY`
5. 视需求支持「高德 / Google 双 Provider」切换

---

## 五、当前优化（无需换服务商）

已实现 **「地点未变化则不重新加载」**：

- 使用地点指纹判断数据是否变化
- 地点不变时保留现有地图实例，不重新请求高德 API
- 可减少加载次数，缓解超时问题
